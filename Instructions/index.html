Purpose & Aim

Product data sheets can be a very useful tool in selling a product but, trying to put all the specific details on a product page can take up just too much room. With this mod, you can add a downloadable PDF to any product. You can briefly explain each product but also provide those interested customers with the full specifications for that particular product.
You specify the text to appear on the product page. This text will show up below the product description with a PDF icon linking to the PDF download file.

Installation
NOTE: This plugin is for Zen Cart 1.5.7c only!

IMPORTANT
Before you begin:
Make a BACKUP of store files and database!
No warranty!
BACKUP made? Okay, then continue...

Follow these steps carefully:

  1. Open the install.sql in the folder SQL with a text editor and copy the content. In your Zen Cart admin panel go to Tools >> Install SQL Patches and paste it in the first block. Follow the directions on the page and press Send to run the installation. This creates the needed database changes.
  2. Download the following file from your Zen Cart installation:
    • /includes/configure.php
    • Open it with a text editor and, after the line define('DIR_FS_CATALOG', '/abcd/');, add define('DIR_WS_PDF_ATTACHMENTS', 'pdf-attachments/');
    • Now upload the changed /includes/configure.php again. Note that this file is normally not writeable, so you will have to change the file permissions to chmod 644 before you upload.
    • After uploading change it back to chmod 444
  3. Download the following file from your Zen Cart installation:
    • YOUR_ADMIN/includes/configure.php where YOUR_ADMIN is the name of your admin directory.
    • Open it with a text editor and, after define('DIR_FS_CATALOG', '/abcd/');, add define('DIR_FS_CATALOG_PDF_ATTACHMENTS', DIR_FS_CATALOG . 'pdf-attachments/');
    • Now upload the changed YOUR_ADMIN/includes/configure.php again. Note that this file is normally not writeable, so you will have to change the file permissions to chmod 644 before you upload.
    • After uploading change it back to chmod 444
  4. In the folder New Files rename the folder YOUR_ADMIN to the name of your admin directory.
    • Upload the folders/files. No existing files will be overwritten.
    • Change the permissions of the folder pdf-attachments to chmod 755 as it needs to be writeable to store the PDF files later.
  5. In the folder Changed Files, rename the folder YOUR_TEMPLATE to the name of your active store template. This can be found by going to Tools >> Template Selection and looking under the Template Directory heading.
  6. Rename the folder YOUR_ADMIN to whatever is actually the name of your admin folder.
  7. NOTE: The files in the Changed Files folder are modified 1.5.7c files. If you have modified any of these files from their standard form in 1.5.7c, you will need to compare the files to make sure any previous changes you made are properly merged with these files prior to uploading them to your site.
  8. When editing a product you will now have a new option to define a description which will be shown later on the product details page for the PDF attachment. And you can upload a PDF file for that product.
    • Working with these PDF attachments should be a familiar process as it mimics the product image section
    • Similar to product images in the /images folder, you can work with subfolders if you wish.
    • Create subfolders in the pdf-attachments folder (they will need chmod 755 as well). As soon as there are subfolders they will be selectable in the "Upload to Directory" dropdown.
    • The Catalog >> Product Types >> Product General >> Layout Settings is where you can turn the display of PDFs on the Product Page on or off. This will turn ALL PDFs on or off.

Note on file types:

  • Only PDF files are allowed for file upload by default. If you want to use it for other file types as well, change the file types in the following file accordingly:
    • YOUR_ADMIN/includes/modules/new_product_preview.php
    • $products_pdf_attachment->set_extensions(array('pdf','PDF'));

Note on file sizes:

  • The same file size restrictions apply to the upload as for the upload of images.
  • By default, the file upload size under Configuration >> Maximum Values >> Maximum File Upload Size is set to 2 MB (2048000).
  • If larger PDFs are to be used, increase this value as desired.

UNINSTALLATION

  1. Use the uninstall.sql in the SQL folder to remove the database changes via Tools >> Install SQL Patches.
  2. Delete all the new files created by ths plugin (those that were in the New Files) from your server.
  3. Replace all the files changed by this plugin with the original files from the backup you made before adding this mod.
